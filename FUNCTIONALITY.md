## Детальное описание реализованного функционала

### 1. Ядро модели клеточного автомата

**Состояния ячеек (рынков):**
- `BLOCKED (0)` — рынок недоступен
- `POTENTIAL (1)` — рынок доступен для анализа
- `EVALUATING (2)` — рынок на этапе оценки
- `ENTERED (3)` — компания вышла на рынок

**Правило перехода:**
```
IF total_score >= ENTRY_THRESHOLD AND state == POTENTIAL:
    state → ENTERED
```

### 2. Функция привлекательности

Взвешенная сумма 6 факторов:

```python
score = (
    0.20 * population +
    0.15 * gdp +
    0.20 * smartphones +
    0.20 * (1 - regulatory) +
    0.15 * (1 - competition) +
    0.10 * infrastructure
)
```

### 3. Эффект соседства

Бонус за присутствие в соседних странах:

```python
bonus = 0.15 * (entered_neighbors / total_neighbors)
```

### 4. Алгоритм симуляции

```
FOR year IN [2015..2024]:
    candidates = []
    FOR country IN countries WHERE state == POTENTIAL:
        IF should_enter_market(country):
            candidates.append(country)
    
    candidates.sort(by=score, descending=True)
    
    FOR country IN candidates[:MAX_ENTRIES_PER_YEAR]:
        country.state = ENTERED
        entry_years[country] = year
```

## Известные ограничения текущей версии

1. **Статические факторы**: значения факторов не меняются во времени
2. **Детерминированность**: модель не учитывает случайные факторы
3. **Ограниченный набор стран**: 5 стран (включая Россию)
4. **Упрощённая топология соседства**: только прямые соседи
5. **Отсутствие валидации входных данных**: предполагается корректность данных
